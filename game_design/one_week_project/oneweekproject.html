周游戏项目

在我大三的第一个学期中，我的专业课要求我们每一周完成一款游戏。在一共12款游戏的快节奏制作中，找到自己所喜欢制作的游戏类型和游戏部分。每周，我们都会被给予一个词作为游戏的主题。在学期结束时，我们在4个月以内做完了截然不同的小游戏。



第一周-Pop（大众的，砰的响声）
第二周-Stack On（积叠）
第三周-Awkward Moment（尴尬的时刻）
第四周-Bounce（弹跳）
第五周-Arcade Game（电玩城游戏）
第六周-Leaving Home（离家）
第七周-One（一）
第八周-Out of Time（时间不够了）
第九周-In View（视野中）
第十周-Level II（第二关）
第十二周-Hero（英雄）



One（一）

×

第七周的主题是一。我的灵感来自于很多游戏中会通过让主角非常的弱小来营造轻松的氛围，主角往往会在玩家意想不到的时候死去，而这点非常的有意思。所以在这个游戏中，主角的血量只有一点，任何小小的陷阱和事物就能夺走他的生命，更别说还需要注意时时都在变化的体力条和饥饿条。在游戏中，我添加了很多陷阱，使得玩家需要处处提高警惕来保证主角的存活。

总结

在这次游戏的制作过程中，我意识到用精神条来搭建场景的难度，因为有时候精神条上的素材并不是同一大小的，所以在切割时，很容易会变的很混乱。在这样的基础下，整理以及命名所有的素材就显得尤为重要。同时，我还尝试了怎样通过代码来触发不同的动画使不同方向的行走动画变得流畅且真实。

Level II（第二关）

作为第十周的主题第二关，我们可以任意选择一个之前做完的游戏来进行二次创作。我选择的则是第九周的游戏，视野中。因为我很喜欢相机，照片这样的游戏元素，而且我认为我第九周的游戏还有很多的改进空间。这周的改动我通过围绕着成为一个好的摄影师需要什么样的要素而设计了三款迷你小游戏。这三款小游戏分别是拍摄一张鸟最接近画面中心的照片，一张画面中鸟的数量尽可能多的照片，和在指定时间内，找到尽可能多的鸟。

点这里跳转游玩页面

负责内容

参与人数：1人（单人项目）
-一个小游戏选择页面和三个小游戏页面的设计和数据共通
-绘制所有的美术素材
-拍照代码
-通过点击来寻找物品代码
-识别图中有多少只鸟的代码

总结

三个小游戏中，因为我在离家那周做了一款类似功能的游戏，所以寻找小鸟对我来说是最简单的。 而另外两个小游戏代码的最终成品不算困难，但过程中也尝试了很多最终没有成功的代码。对我来说，其实很多代码上的问题并没有被解决，比如在一开始的代码中哪怕鸟被树挡住，也会被算进拍到的数量，随机生成鸟还是实其有流畅的动画和随机的停留时间对目前的我来说还是太过于困难。但我对于最终成品还是比较满意的，尽管帧数不高，但是鸟的动画看起来还是比较流畅的，我也找到了其他方法来计算画面中鸟的数量。

如果我有更多的时间，应该会往里加入一些剧情，经过我的设计是围绕着成为一名好的摄影师来制作的，但是玩家并不知道这一点，如果有剧情把所有游戏串起来应该会很有意思。

Hero（英雄）

这是我这个学期的最终项目，所以和其他的周项目不同，这个游戏的制作周期有足足三个星期。而且我们也有两个关于主题的选择，分别是英雄和鸡蛋。作为额外的要求，如果选择英雄这个主题，游戏中则需要使用到老师提供的三个小手办。我选择英雄这个主题因为我觉得小手办们非常的可爱，而且英雄这个主题本身也有更多的发挥空间。我一直像做一个有背包低通的有更多选择的解谜游戏，这三周正好是一个很好的机会让我实现这样一个完成度更高，内容更多的游戏想法。

My game idea for this one is that three characters represent three different abilities, breaking things, time passing, and getting things from far away.

总结

I think I finished the majority of what I want to achieve in this game, there are multiple endings, a relatively free explore world, and a puzzle solving game that is longer and a little more complicated.

There are still bugs and errors I have to fix, but I did not have enough time like order of sprites, sometimes the move button is not working or the ability UI did not stay at the position it should be. But because it is not an easy fix, and it does not affect the overall gameplay, I did not fix it right away.

I like this game, like the art style and overall dragging gameplay. It is like a mobile game, easy to play. Moreover, it is a puzzle game which is the kind of game I enjoy making and playing by myself. Although I think the puzzle itself can be more logical and harder, it needs more time within the planning stage. So if I have another chance, I will spend more time planning out the whole game. But i think it is in a pretty good place for three weeks game.
